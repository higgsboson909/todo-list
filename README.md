## Design
- todos are going to be objects
    create dynamically
    using factories/constructors(classes)

- todo-items should have
    - title
    - description
    - due Date
    - priority
    - may include notes / checklist

- app should have
    default 'today' project to create simple tasks......
    can create projects
        inside projects, they should have todos

- look of UI
    view all projects
    view all todos in each project
    expand a single todo to see edit its details
    delete a todo
